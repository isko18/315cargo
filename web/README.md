# 315CARGO — админ-панель (React + Vite + TS)

Веб-панель для операторов/владельца: **приём посылок по штрих-коду** и **обзор
всех карго**. Бьёт в готовое API бэкенда.

## Запуск

```bash
cd web
cp .env.example .env      # при необходимости поменяй VITE_API_BASE
npm install
npm run dev               # http://localhost:5173
npm run build             # прод-сборка в dist/
```

## Что внутри

| Страница | API | Кому |
|---|---|---|
| `/scan` — Приём по штрих-коду | `POST /api/parcels/scan/`, `POST /api/parcels/{id}/assign/` | оператор карго / владелец |
| `/issue` — Выдача клиенту | `GET /api/parcels/?client_code=`, `POST /api/parcels/scan/` (status=issued) | оператор карго / владелец |
| `/overview` — Все карго | `GET /api/admin/overview/` | владелец (супер-админ) |
| `/login` | `/api/cargo-companies/`, `/api/auth/send-code/`, `/api/auth/verify-code/` | все |

**Выдача:** код клиента вводят вручную, **аппаратным сканером** (печатает код +
Enter) или **камерой** — кнопка «📷 Сканировать QR» наводится на QR клиента
(QR кодирует его `client_code`). Дальше грузятся посылки клиента → выбираешь →
«Выдать» переводит каждую в статус «Выдан» (`scan` с `status=issued`). Уже
выданные подсвечены и недоступны.

> Камера в браузере работает только по **HTTPS** (или на `localhost`). Для
> продакшена панель должна раздаваться по https, иначе доступа к камере не будет.

## Вход

- **Оператор карго / карго-админ** — по SMS: выбрать карго, телефон, код.
- **Владелец (супер-админ без карго)** — вход по SMS не подходит (у него нет
  карго), поэтому на экране входа есть «войти по токену»: вставь `access` JWT
  (например из `/api/auth/` или полученный иным способом). Обзор всех карго
  (`/overview`) требует прав супер-владельца.

## Приём по штрих-коду

- Поле ввода в фокусе — аппаратный сканер печатает код и жмёт Enter →
  автоматически уходит `scan`. Работает и с ручным вводом.
- Статус по умолчанию — «Прибыл на склад в Китае» (меняется в селекте).
- Бэкенд сам: если по треку есть заказ клиента — привяжет; если нет — создаст
  «без клиента». Непривязанные строки можно тут же привязать по коду клиента.
- Супер-админ без карго указывает `Карго ID`; оператор карго — оставляет пустым.

## Деплой

`npm run build` → статика в `dist/`. Раздать любым статик-сервером (nginx) или с
того же домена, что API. `VITE_API_BASE` должен указывать на бэкенд
(по умолчанию `https://315cargo.webtm.ru`).
